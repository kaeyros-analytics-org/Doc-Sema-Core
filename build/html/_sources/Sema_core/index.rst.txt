Fonctionnalités de Sema Core
============================

Cette section décrit les principales fonctionnalités de Sema Core et explique comment les utiliser étape par étape.

Chaque module est présenté avec :

- Son objectif ;
- Les actions disponibles ;
- Les étapes d'utilisation ;
- Les points de contrôle ;
- Les emplacements prévus pour insérer les captures d'écran.

Vue d'ensemble des modules
---------------------------

.. list-table::
   :header-rows: 1
   :widths: 25 75

   * - Module
     - Rôle principal
   * - Tableau de bord
     - Suivre les indicateurs, les campagnes, les conversations et les performances.
   * - Chat
     - Suivre les conversations et gérer les interactions client.
   * - Scenario Builder
     - Construire des parcours conversationnels automatisés avec des nœuds.
   * - Flow Builder
     - Créer des parcours structurés, des formulaires ou des flow guidés.
   * - Contacts
     - Créer, importer, exporter, rechercher et qualifier les contacts.
..   * - Paramètres
..     - Configurer les accès, préférences, intégrations et éléments globaux.
..   * - Messages en masse
..     - Préparer et envoyer des communications groupées.
..   * - Catalogue
..     - Gérer les produits utilisés dans les conversations et catalogues interactifs.
..   * - Programmes
..     - Gérer les fonctionnalités d'engagement comme fidélité, tombola ou événements.

Parcours recommandé pour un nouvel utilisateur
----------------------------------------------

**1.** Commencer par le :doc:`tableau_de_bord` pour comprendre l'activité.

**2.** Ouvrir le :doc:`chatbot` pour consulter les conversations.

**3.** Lire le :doc:`scenario_builder` avant de modifier un scénario.

**4.** Consulter le :doc:`flow_builder` pour les parcours structurés.

**5.** Vérifier les :doc:`contacts` que vous avez créés.

.. **6.** Configurer les :doc:`parametres` uniquement si vous avez les droits nécessaires.

.. **7.** Utiliser :doc:`messages_masse` pour les campagnes groupées.

.. **8.** Vérifier le :doc:`catalogue` si les conversations affichent des produits.

Bon à savoir
------------

.. tip::

   Avant toute modification importante, prenez une capture de l'état actuel et notez le nom du scénario ou du paramètre que vous allez modifier. Cela facilite les vérifications avec l'équipe.
